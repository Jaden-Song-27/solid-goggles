# SmartIME — 产品路线图

## v0.1 — 核心体验验证 ✅ 当前版本

**目标**：验证"后悔模式 + 可编程皮肤"的产品假设，获得第一批种子用户反馈。

| 模块 | 内容 | 状态 |
|------|------|:--:|
| 拼音引擎 | 音节切分 + 5200 汉字字库 + 频率自学习 | ✅ |
| 后悔模式 | 2 秒内 Ctrl+Z 回退重选，连续后悔 | ✅ |
| 皮肤系统 | 5 套内置皮肤 + CSS 变量引擎 + 动画系统 | ✅ |
| 皮肤编辑器 | WYSIWYG 可视化编辑 + 导入/导出 | ✅ |
| 快捷指令 | 6 个内置指令 + 自定义指令 + 模板占位符 | ✅ |
| 系统集成 | 双 Ctrl 呼出、系统托盘、开机自启 | ✅ |
| 文本注入 | Windows SendInput 原生注入 | ✅ |

---

## v0.3 — 词库与体验增强

**目标**：缩小与主流输入法的词库差距，提升基础输入体验。

| 优先级 | 功能 | 说明 |
|:--:|------|------|
| P0 | 常用词库接入 | 扩展为 5 万+ 中文词汇，支持词语级候选 |
| P0 | 中英混输 | 自动识别英文单词，不用切换输入模式 |
| P1 | 云词库（可选） | 可配置的联网词库更新，默认关闭 |
| P1 | 自定义短语 | 用户可定义缩写→短语映射（如 `dz` → 地址） |
| P2 | 颜文字/Emoji | 快捷输入颜文字和 Emoji |
| P2 | 智能纠错 | 常见拼写错误自动纠正（zogn→中） |

---

## v0.5 — 智能预测引擎

**目标**：基于本地 n-gram 语言模型，显著提升候选词排序准确率。

| 优先级 | 功能 | 说明 |
|:--:|------|------|
| P0 | 2-gram/3-gram 词频 | 基于大规模语料的本地二元/三元语言模型 |
| P0 | 上下文感知排序 | 根据光标前 10-30 字调整候选词排序 |
| P0 | 个性化语言模型 | 在本地训练用户专属的小型语言模型 |
| P1 | 行业词库 | 编程/医学/法律等专业词库包 |
| P2 | macOS 支持 | 验证并适配 macOS 平台 |

---

## v1.0 — AI 大模型接入

**目标**：成为真正意义上的"AI 输入法"，接入大模型实现智能写作辅助。

| 优先级 | 功能 | 说明 |
|:--:|------|------|
| P0 | LLM 句子补全 | 输入几个字 → AI 预测整句（本地模型优先） |
| P0 | AI 写作辅助 | 翻译/润色/扩写/缩写 一键操作 |
| P1 | 上下文理解 | 读光标前的文档内容，提供更智能的建议 |
| P1 | 多模态输入 | 截图识别文字、语音转文字 |
| P2 | Linux 支持 | 适配 Linux 桌面环境 |
| P2 | 插件系统 | 开放的插件 API，第三方开发者可扩展 |

---

## 长期愿景

> **SmartIME 不是另一个输入法，而是一个桌面端的 AI 交互入口。**

当用户在任何一个应用里打字时，SmartIME 是他们与 AI 交互的第一触点。它理解上下文、知道用户意图、可以执行操作——比 Siri 精准，比 ChatGPT 轻量，比 Copilot 更贴近"写"这个动作为中心的工作流。

### 关键词
- **Input as Interface** — 输入即界面
- **Local-first AI** — 隐私优先的本地智能
- **Programmable IME** — 开发者友好的输入法平台
