# SmartIME — 用户画像

## 核心用户 1：效率型知识工作者

### 典型人物：小李，28 岁，后端工程师

**日常场景**：
- 每天写代码 6 小时，其中穿插大量中文沟通（PR 描述、技术文档、即时通讯）
- 频繁在中英文之间切换，写技术文档时经常需要输入特殊符号和公式
- 最痛恨的事：写了一大段中文后发现中间有个错别字，需要删掉后半段重打

**使用 SmartIME 的动机**：
- 后悔模式：写代码注释/文档时打错字能快速回退修正
- `/calc` 命令：随手计算，不离开编辑器
- 可定制的极简皮肤：不干扰代码编辑器的视觉环境

**决策路径**：
技术博客看到 → 下载试用 → 后悔模式"真香" → 推荐给同事

---

## 核心用户 2：个性化工具控

### 典型人物：小陈，24 岁，UI 设计师 / 数字创作者

**日常场景**：
- 喜欢折腾工具的外观和交互
- 所有桌面软件都经过了视觉定制（Rainmeter、壁纸引擎等）
- 在社交平台分享自己的桌面配置

**使用 SmartIME 的动机**：
- 皮肤编辑器：像写 CSS 一样定制输入法外观
- 5 套内置皮肤的视觉冲击力
- 导出皮肤分享到社区，获得认同感

**决策路径**：
社交媒体看到别人晒皮肤 → 被视觉效果吸引 → 下载 → 自己做一套 → 发帖

---

## 核心用户 3：AI 产品早期尝鲜者

### 典型人物：小王，30 岁，产品经理 / AI 训练师

**日常场景**：
- 关注 AI/工具类产品动态，习惯尝鲜新软件
- 工作中需要快速切换不同 AI 工具（ChatGPT → 文档 → 原型工具）
- 有写产品分析/评测的习惯

**使用 SmartIME 的动机**：
- "AI 输入法"的概念吸引力
- 快捷指令可以被扩展成工作流
- 关注 v1.0 的 LLM 接入计划

**决策路径**：
产品社区看到文章 → 体验 → 写评测反馈 → 成为种子用户

## 非目标用户

| 用户类型 | 为什么不适合 SmartIME (v0.1) |
|----------|---------------------------|
| 老年用户（60+） | 需要大字体/手写/语音，不是 SmartIME 的优势 |
| 纯移动端用户 | v0.1 仅支持 Windows 桌面 |
| 只需要语音输入的用户 | 未实现语音功能 |
| 对输入法零兴趣的普通用户 | 不会主动更换输入法 |

---

## 用户增长策略

1. **开发社区起量**：先在 V2EX、GitHub、掘金等技术社区推广，获得种子用户
2. **"后悔模式"做为钩子**：这是最容易被记住和传播的功能点
3. **皮肤社区化**：开放皮肤导入导出格式，鼓励用户创作和分享
4. **AI 输入法的话题红利**：在人人都是产品经理、PMCAFF 发布产品分析文章
